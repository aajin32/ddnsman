# ddnsman — 把家里服务器变成稳定的公网服务

你的宽带每次重启 IP 就变了？外网访问家里服务器总连不上？

ddnsman 能自动检测公网 IP 变化并更新阿里云 DNS 记录，让域名始终指向最新的 IP。

## 一、这东西是干什么的

你的网络结构大概是这样：

```
你的家宽网络（有公网 IP，但不是固定的）
    │
    ├── 你的路由器（负责拨号上网）
    │       └── 内网一台服务器（或 Docker）
    │               └── 运行 ddnsman（24 小时开机）
    │                       └── 发现 IP 变了 → 自动更新阿里云 DNS
    │
别人访问你的域名 ──→ 阿里云 DNS ──→ 你的最新公网 IP ──→ 路由器 ──→ 你的服务器
```

ddnsman 就干一件事：**盯住你的公网 IP，变了就自动更新 DNS 记录**。别人始终能通过域名访问到你。

## 二、你需要准备什么

### 必须满足 3 个条件

| # | 条件 | 说明 |
|---|------|------|
| 1 | **你的宽带确实有公网 IP** | 电信/联通一般都带，不确定就问宽带客服："我家有没有公网 IPv4？" 如果他说没有，那你用不了 |
| 2 | **你能登录路由器管理页面** | 通常是浏览器打开 `192.168.1.1` 或 `192.168.0.1`，账号密码在路由器背面贴纸上 |
| 3 | **你有一个阿里云域名** | 比如 `example.com`，在阿里云买的那种，或把域名 DNS 托管到阿里云的 |

### 一台 24 小时开机的服务器

因为 IP 随时可能变，ddnsman 需要一直在后台跑着。可以是：

- 一台**不关机的旧电脑/树莓派**（装 Ubuntu Server）
- **云服务器**（比如阿里云 ECS 最便宜那档）
- **路由器上跑 Docker**（如果你的路由器支持）
- VPS、NAS 等都行

> 不要在你日常用的笔记本上跑——你合上盖子 IP 就失联了。

### 一个阿里云 RAM 子账号的 AccessKey

别用主账号！原因见下文。配置方法在 **[第五章](#五ram-权限配置)**，建议先跳到那看一眼，配好了再回来。

## 三、跟我一步步操作

### 第 1 步：把脚本放到你的服务器上

> 如果你不知道什么是 SSH，看这里：SSH 就是远程登录你服务器的方式。
> - **Windows 用户**：下载 [PuTTY](https://www.putty.org/)，主机名填你的服务器 IP，端口 22，点打开，输入用户名密码
> - **Mac / Linux 用户**：打开终端，输入 `ssh root@你的服务器IP`，回车，输入密码

登录到你的服务器后，任选一种方式把脚本弄到服务器上：

**方式一：直接用 wget 下载（推荐，最快）**

```bash
wget https://raw.githubusercontent.com/aajin32/ddnsman/main/python/ddnsman.py
chmod +x ddnsman.py
```

**方式二：git clone 整个项目**

```bash
git clone https://github.com/aajin32/ddnsman.git
cd ddnsman/python
chmod +x ddnsman.py
```

### 第 2 步：直接运行，自动装环境

```bash
./ddnsman.py
```

第一次运行会看到类似下面的输出：

```
⏳ 正在安装依赖: aliyun-python-sdk-alidns, questionary, requests
```

**这是正常的**，脚本在自动安装需要的组件。

> ⚠️ **注意**：如果系统没有 pip，脚本会自动安装。这时会弹一次 `sudo` 密码输入框：
>
> ```
> [sudo] 你的用户名 的密码：
> ```
>
> **输入你的登录密码**（输入时屏幕不会显示字符，这是正常的），回车继续。
>
> 如果脚本发现 pip 被系统限制（新版 Ubuntu 常见），还会自动第二次尝试 `sudo pip install`，同样会弹一次 sudo 密码框。**全程只需看着就行**，脚本会自动处理。

看到下面就是装好了：

```
✅ 依赖安装完成
```

然后会进入**配置引导**：

```
请配置阿里云 AccessKey（RAM 子账号，最小权限）

? AccessKey ID:   <-- 输入你的 AccessKey ID
? AccessKey Secret:   <-- 输入你的 AccessKey Secret（输入时不显示，正常）
? 默认域名（如 example.com）: example.com
```

填完后会自动验证你的 AccessKey 是否可用。看到下面就是成功了：

```
✅ 配置验证通过
```

### 第 3 步：新增一条 DNS 记录

接着你会看到**主菜单**：

```
主菜单  [域名: example.com]
  1. 查看/管理解析记录
  2. 新增解析记录
  3. 切换域名
  4. 重新配置 AccessKey
  5. DDNS 设置
  6. 退出
```

按键盘 **↓** 键移动到「2. 新增解析记录」，按回车。

然后按提示一步步来：

```
? 主机记录（如 www、@）: myhome          ← 你想用什么域名？填 myhome 就是 myhome.example.com
? 记录类型: → A                          ← 选 A（按 ↑↓ 然后回车）
? 是否自动获取本机公网 IP？ Yes           ← 选 Yes，脚本会帮你查
```

这时会有一个**倒计时**：

```
⏳ 正在获取公网 IP... 10秒
```

正常的话几秒内就能查到：

```
✅ 获取到公网 IP: 1.2.3.4
```

然后关键一步：

```
? 是否开启定时更新（检测 IP 变化自动同步）？ Yes   ← 选 Yes！
```

最后确认：

```
? 确认新增: myhome.example.com A -> 1.2.3.4 TTL=600 [定时更新]
```

看到这个就是搞定了：

```
✅ 已新增: myhome.example.com A -> 1.2.3.4
✅ 已添加 crontab 定时任务，每 5 分钟检测一次
```

**DDNS 功能已自动开启**，脚本每 5 分钟自动检查一次 IP，变了就更新。

### 第 4 步：在路由器上做端口转发

这一步的目的是：**别人访问你的域名时，能穿透路由器找到你的服务器**。

> 这一步不是本脚本的功能，但**不做的话外网访问不了你的服务**。

具体方法（不同路由器界面不同，但思路一样）：

1. 浏览器打开你的**路由器管理页面**（通常是 `192.168.1.1`）
2. 找到 **端口转发**（Port Forwarding）或 **虚拟服务器**（Virtual Server）选项
3. 添加一条规则，比如你要跑 Web 服务：

   | 项目 | 填写 |
   |------|------|
   | 外网端口 | 80（或别的端口） |
   | 内网 IP | 你服务器的内网 IP（比如 192.168.1.100） |
   | 内网端口 | 服务实际运行的端口 |
   | 协议 | TCP |
   | 状态 | 启用 |

4. 保存生效

现在别人访问 `myhome.example.com:80` 就能到你的服务器了。

### 第 5 步（可选）：检查 DDNS 是否正常工作

```bash
# 手动执行一次 IP 检测
./ddnsman.py --watch
```

正常输出：

```
[ddnsman] 2026-05-12 14:30:00  公网 IP: 1.2.3.4
  myhome.example.com: IP 无变化 (1.2.3.4)
```

## 四、日常使用说明

### 主菜单

```
主菜单  [域名: example.com]
  1. 查看/管理解析记录    → 列出所有记录，选一条后可修改或删除
  2. 新增解析记录          → 按指引加一条新记录
  3. 切换域名              → 换一个域名管理
  4. 重新配置 AccessKey    → 换一组 AccessKey
  5. DDNS 设置             → 管理定时更新、改检测频率
  6. 退出
```

### 常用的菜单操作

**查看已有记录：** 主菜单选 1 → 选一条记录 → 可以点修改或删除

**删除某条 DDNS 跟踪：** 主菜单选 1 → 选记录 → 选「定时更新 [关闭]」

**修改检测频率（默认 5 分钟）：** 主菜单选 5（DDNS 设置）→ 选「修改检测间隔」

**彻底停掉 DDNS：** 主菜单选 5 → 选「卸载 crontab」

### 想重新开始怎么办

```bash
rm -f ~/.alidns_config.json
./ddnsman.py   # 会重新引导配置
```

## 五、RAM 权限配置

### 本脚本用到的接口

| 接口 | 干什么用的 |
|------|-----------|
| `DescribeDomainRecords` | 查你的域名有哪些记录 |
| `AddDomainRecord` | 新增一条记录 |
| `UpdateDomainRecord` | 修改记录的值 |
| `DeleteDomainRecord` | 删除记录 |

### 创建子账号（图文指引）

1. 打开 [RAM 控制台](https://ram.console.aliyun.com/)
2. 左侧点「用户」→「创建用户」
3. 登录名称填 `ddnsman`，勾选 **OpenAPI 调用访问**（这是给 AccessKey 用的）
4. 点确定，会生成一串 AccessKey ID 和 AccessKey Secret，**复制保存好**（关了就看不了了）
5. 再点「添加权限」
6. 如果你嫌麻烦，直接搜 **`AliyunDNSFullAccess`** 勾上 → 确定

### 最小权限（更安全，推荐）

你也可以按最下面步骤来，创建自定义策略：

1. 左侧点「权限策略」→「创建策略」
2. 选「脚本编辑」，把下面内容粘贴进去：

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "alidns:DescribeDomainRecords",
        "alidns:AddDomainRecord",
        "alidns:UpdateDomainRecord",
        "alidns:DeleteDomainRecord"
      ],
      "Resource": "*"
    }
  ]
}
```

3. 策略名称填 `ddnsman-policy`，点确定
4. 回到用户列表，找到刚才创建的 `ddnsman` 用户
5. 「添加权限」→ 搜索刚才创建的策略 → 勾上 → 确定

## 六、配置说明

配置文件自动生成在 `~/.alidns_config.json`，内容：

```json
{
  "access_key_id": "LTAI5txxx",
  "access_key_secret": "xxxxxxxxxx",
  "domain": "example.com",
  "ttl": 600,
  "ddns_records": ["myhome"],
  "ddns_interval": 5
}
```

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `ttl` | DNS 记录的缓存时间（秒），不懂别改 | 600 |
| `ddns_records` | 哪些主机记录需要自动检测更新 | `[]` |
| `ddns_interval` | 每隔几分钟检测一次 IP | 5 |

IP 缓存文件 `~/.alidns_ip_cache.json`，记录上次的 IP 值，干活用的，不用管它。

---

## 七、注意事项

- **绝对不要用阿里云主账号的 AccessKey**。去 RAM 创建个子账号，给最小权限就行
- 配置文件 `~/.alidns_config.json` 脚本自动设为只有你能读（权限 600），别手动改它的权限
- 公网 IP 获取：依次尝试 `ipinfo.io/ip` → `ifconfig.me` → `icanhazip.com` → `checkip.amazonaws.com`，一个不行换下一个
- 卸载 cron 任务时**脚本不会删行**，只是加 `#` 注释掉，留了后悔药
- **删除 DNS 记录不可恢复**，脚本有二次确认弹窗，看清楚再点

## 八、常见问题

### 怎么看我的宽带是不是真的有公网 IP？

方法一：登录路由器管理页面 → 状态/WAN 口信息 → 看显示的 IP 地址。然后打开手机上淘宝搜 "IP"，对比两个是不是一样。一样就是有公网 IP。

方法二：在你的服务器上执行 `curl ipinfo.io/ip`，它会显示你的公网 IP。如果这个 IP 和路由器的 WAN 口 IP 一致，就是公网。

### 装到一半弹出 sudo 是什么？我该输什么？

这是系统在问你的密码。**输入你的登录密码后回车**，输入时屏幕不会显示任何字符（连星号都没有），这是正常的安全设计。

### 执行 ./ddnsman.py 报错说没权限

先执行 `chmod +x ddnsman.py` 再重新运行。

### 运行后菜单是乱码或不清晰

检查你的终端是否支持 UTF-8，执行：

```bash
locale
```

确保输出中有 `LANG=zh_CN.UTF-8` 或类似的 UTF-8 编码。

### DDNS 开启了，但域名还是解析到旧的 IP

手动执行一次检测看看：

```bash
./ddnsman.py --watch
```

如果显示「IP 无变化」，说明当前 IP 和阿里云上记录的一样，是正常的。如果显示「更新失败」，检查 AccessKey 权限。

也可以登录[阿里云云解析控制台](https://dns.console.aliyun.com/) 检查记录值是否已更新。

### 换了服务器，怎么迁移？

把旧服务器的 `~/.alidns_config.json` 文件复制到新服务器的相同位置，然后在新服务器上直接运行 `./ddnsman.py` 即可。

### 不小心删了配置文件怎么办？

删了没事，重新运行 `./ddnsman.py` 会重新引导输入。

### crontab 报错 "command not found"

说明系统没装 cron，执行：

```bash
sudo apt-get install cron -y
```
